# Backend Build Warning Cleanup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Eliminate every `cargo build`/`cargo check` warning in the `backend/` workspace by removing genuinely dead code (an orphaned function, a write-only DB column and its dead reads) and fixing an architectural bug (duplicate module compilation in the `crawler` binary) — without changing any observable behavior.

**Architecture:** No new abstractions. Four independent, surgical removals/fixes across `api`, `worker`, and `crawler`, plus one migration edit. Each task leaves the workspace fully compiling and all tests passing.

**Tech Stack:** Rust workspace (Axum API, sqlx with compile-time-checked queries, Postgres).

## Global Constraints

- After touching any `sqlx::query!`/`query_as!` macro, run `bash backend/prepare-sqlx.sh` to regenerate `.sqlx/` before the next build/test — offline builds (`SQLX_OFFLINE=true`, used by `test.sh`) will fail on a stale cache.
- Run `cd backend && cargo fmt --all` before considering any task done.
- The `ranking_set_results.upset_factor` migration column may be edited directly in `backend/migrations/001_initial.sql` rather than via a new migration — there is no production database yet (user-authorized in the design spec).
- Full backend suite: `bash backend/test.sh` from the repo root (spins up an ephemeral Postgres container, sets `SQLX_OFFLINE=true`, runs `cargo test --workspace`). Pass extra args straight to `cargo test`, e.g. `bash backend/test.sh -p api`.
- Design spec: `docs/superpowers/specs/2026-07-09-backend-warning-cleanup-design.md`. This plan implements **Part 1 only**. Part 2 (the DQ-detection correctness gap in `crawler`) is an explicitly out-of-scope follow-up — do not touch `crawler::scraper::is_dq()`, `extract_scores()`, or the `state`/`displayScore`/`stats.score.value` handling in this plan.

### Task 4: Fix `crawler` duplicate bin/lib module compilation

**Files:**
- Modify: `backend/crates/crawler/src/main.rs`

**Interfaces:** `crawler::cli::Config` and `crawler::scraper::run` are already `pub` in `lib.rs` (`pub mod cli; pub mod scraper;` — confirmed in `lib.rs`, unchanged). No signature changes; this only changes how `main.rs` reaches them.

**Why:** `main.rs` currently redeclares `mod api; mod api_types; mod cli; mod db; mod scraper;`, which are the exact same module tree `lib.rs` already declares as `pub mod`. This makes Cargo compile every one of those source files twice — once for the `crawler` lib target, once for the `crawler` bin target, as two separate crate roots — which is why most crawler dead-code warnings are reported twice (once per target). `main.rs` only ever references `cli::Config` and `scraper::run` directly.

- [ ] **Step 1: Replace the `mod` redeclarations with a lib import**

Change the top of `main.rs` from:

```rust
use clap::Parser;
use std::sync::Arc;
use std::sync::atomic::{AtomicBool, Ordering};
use tokio::signal::unix::{SignalKind, signal};

use cli::Config;

mod api;
mod api_types;
mod cli;
mod db;
mod scraper;
```

to:

```rust
use clap::Parser;
use std::sync::Arc;
use std::sync::atomic::{AtomicBool, Ordering};
use tokio::signal::unix::{SignalKind, signal};

use crawler::{cli, scraper};
```

- [ ] **Step 2: Update the two remaining references**

`main()` currently does `let config = Config::parse();` — change to:

```rust
    let config = cli::Config::parse();
```

`scraper::run(&config, &pool, &shutdown)` inside the `tokio::select!` block is already qualified as `scraper::run(...)` — no change needed there, it now resolves through the new `use crawler::{cli, scraper};` import.

- [ ] **Step 3: Verify the duplicate compilation is gone**

Run: `cd backend && SQLX_OFFLINE=true cargo check -p crawler --all-targets 2>&1 | grep "generated"`
Expected: warning counts roughly halve compared to before (previously `bin "crawler"` reported 19 warnings and `bin "crawler" test` reported 18 duplicates on top of the lib's own warnings; after this fix each dead field/function should be reported once per underlying warning, not once per target). The remaining warnings are all in `api_types.rs` and are explicitly deferred per the design spec (Part 1 §4) — this task does not eliminate them, only stops double-counting them.

Run: `bash backend/test.sh -p crawler`
Expected: `PASS` — crawler's `#[sqlx::test]` integration tests and unit tests (`is_dq`, `extract_scores`, etc.) behave identically; this is a pure module-resolution change with no logic difference.

- [ ] **Step 4: Commit**

```bash
git add backend/crates/crawler/src/main.rs
git commit -m "fix(crawler): stop double-compiling lib modules from the bin target

main.rs redeclared the same module tree lib.rs already exposes as pub,
```

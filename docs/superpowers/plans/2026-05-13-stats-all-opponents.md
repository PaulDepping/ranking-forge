# Stats: All Opponents + Game Scores — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend `GET /projects/{id}/stats` to include sets against non-project opponents, and return `winner_score`/`loser_score` for every set.

**Architecture:** Change the SQL in `get_stats` from INNER JOINs on both player sides to LEFT JOINs, filtering to sets where at least one entrant is a project player. Add score columns to `SetRecord`. Update the frontend type and the Svelte component to render scores from the player's perspective.

**Tech Stack:** Rust/Axum/sqlx (backend), SvelteKit/TypeScript/Vitest (frontend)

    let outside_e =

    // Outsider beats Alice

    let entries = stats.as_array().unwrap();

    assert_eq!(alice["wins"], json!([]));
    assert_eq!(alice["losses"].as_array().unwrap().len(), 1);
    assert_eq!(alice["losses"][0]["opponent_name"], "Outsider");
}
